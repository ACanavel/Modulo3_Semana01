Funciones en JavaScript

Las funciones son bloques de código que realizan una tarea específica y se pueden reutilizar en todo el programa. Es esencial entender cómo declararlas, invocarlas y utilizarlas para escribir código eficiente y organizado.

Declaración de Funciones

Una función en JavaScript se puede declarar de varias formas:

  • Función clásica: Es la forma más común de declarar una función.
  • Función anónima: Funciones que no tienen nombre y se pueden asignar a variables.
  • Funciones flecha (Arrow Functions): Introducidas en ES6, son una sintaxis más concisa para escribir funciones.
            function saludar(nombre) {
                return `Hola, ${nombre}!`;
            }

            const despedir = function(nombre) {
                return `Adiós, ${nombre}!`;
            };

            const sumar = (a, b) => a + b;

            console.log(saludar('Estudiante')); // Salida: Hola, Estudiante!
            console.log(despedir('Estudiante')); // Salida: Adiós, Estudiante!
            console.log(sumar(5, 3)); // Salida: 8

Funciones de Orden Superior

Las funciones de orden superior son funciones que pueden aceptar otras funciones como argumentos o devolverlas como resultados. Estas son fundamentales en la programación funcional y se usan comúnmente en métodos como map, filter y reduce.

            function aplicarOperacion(arr, operacion) {
                return arr.map(operacion);
            }

            const duplicar = x => x * 2;
            console.log(aplicarOperacion([1, 2, 3], duplicar)); // Salida: [2, 4, 6]

Callbacks

Un callback es una función que se pasa a otra función como argumento y se ejecuta después de que se haya completado alguna operación. Los callbacks son esenciales para manejar la asincronía en JavaScript.

            function saludar(nombre, callback) {
                console.log(`Hola, ${nombre}`);
                callback();
            }

            function despedir() {
                console.log('Adiós!');
            }

            saludar('Estudiante', despedir);
            // Salida:
            // Hola, Estudiante
            // Adiós!

Control de Flujo

El control de flujo permite dirigir el orden de ejecución de las instrucciones en un programa. En JavaScript, utilizamos estructuras como if, else, for, y switch para tomar decisiones y repetir tareas.

Condicionales

Las estructuras condicionales permiten que tu programa tome decisiones en tiempo de ejecución:

  • if: Evalúa una condición y, si es verdadera, ejecuta un bloque de código.
  • else: Se ejecuta si la condición if no es verdadera.
  • else if: Permite evaluar múltiples condiciones.
  • switch: Evalúa una expresión y ejecuta el bloque de código que coincide con su valor.
            let dia = 'lunes';

            if (dia === 'lunes') {
                console.log('Es lunes, inicio de la semana.');
            } else if (dia === 'viernes') {
                console.log('Es viernes, casi fin de semana.');
            } else {
                console.log('Es otro día de la semana.');
            }

            let fruta = 'manzana';

            switch(fruta) {
                case 'manzana':
                    console.log('Es una manzana.');
                    break;
                case 'banana':
                    console.log('Es una banana.');
                    break;
                default:
                    console.log('Es otra fruta.');
            }

Bucles

Los bucles permiten repetir una acción varias veces. En JavaScript, hay varios tipos de bucles:

  • for: Itera un número específico de veces.
  • while: Itera mientras una condición sea verdadera.
  • do...while: Ejecuta primero, luego verifica la condición.
            for (let i = 0; i < 5; i++) {
                console.log(`Iteración ${i}`);
            }

            let j = 0;
            while (j < 3) {
                console.log(`While loop, iteración ${j}`);
                j++;
            }

            let k = 0;
            do {
                console.log(`Do...while loop, iteración ${k}`);
                k++;
            } while (k < 2);

Manejo de Errores con try-catch

El bloque try-catch se utiliza para manejar errores que puedan ocurrir durante la ejecución del código. Es una forma efectiva de prevenir que un error detenga la ejecución del programa.

            try {
                let resultado = 10 / 0;
                if (resultado === Infinity) {
                    throw new Error('División por cero');
                }
            } catch (error) {
                console.error('Error:', error.message);
            }
            // Salida: Error: División por cero

Evaluación de Condiciones Lógicas

Las condiciones lógicas se utilizan para tomar decisiones en base a expresiones booleanas. Los operadores lógicos como && (AND), || (OR), y ! (NOT) son fundamentales para evaluar estas condiciones.

Operadores Lógicos

Los operadores lógicos permiten combinar varias condiciones y evaluarlas en conjunto. Aquí tienes los más comunes:

  • && (AND): Devuelve verdadero si ambas condiciones son verdaderas.
  • || (OR): Devuelve verdadero si al menos una de las condiciones es verdadera.
  • ! (NOT): Invierte el valor de una condición.
        let tieneLicencia = true;
        let sabeConducir = false;
        
        if (tieneLicencia && sabeConducir) {
            console.log('Puedes conducir el coche');
        } else {
            console.log('No puedes conducir el coche');
        }
        
        let esNovato = !sabeConducir;
        console.log(esNovato); // Salida: true

Operador Ternario

El operador ternario es una forma concisa de realizar evaluaciones condicionales en una sola línea. Es ideal para situaciones en las que necesitas asignar un valor basado en una condición simple.

        let esMayorDeEdad = true;
        let mensaje = esMayorDeEdad ? 'Puedes votar' : 'No puedes votar';
        console.log(mensaje); // Salida: Puedes votar

Cortocircuito Lógico

El cortocircuito ocurre cuando JavaScript deja de evaluar expresiones lógicas porque ya ha determinado el resultado:

  • && (AND): Si la primera condición es falsa, no evalúa la segunda.
  • || (OR): Si la primera condición es verdadera, no evalúa la segunda.
        let estaAutenticado = true;
        let tieneAccesoAdmin = false;
        
        console.log(estaAutenticado && tieneAccesoAdmin); // Salida: false (no evalúa tieneAccesoAdmin)
        console.log(estaAutenticado || tieneAccesoAdmin); // Salida: true (no evalúa tieneAccesoAdmin)
